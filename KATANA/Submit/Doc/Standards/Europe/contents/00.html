Dreamcast Software Development Standards/Ver. 2.00EU

The following indicators are used to denote particular conditions of the standards: Required, Recommended, Standardised, and Prohibited

The meanings of these symbols are as follows:


Required: These items must be strictly observed during Dreamcast application creation.

Violating one of these items will cause your product to be refused ROM Release.


Recommended: These items are recommended from the perspective of user friendliness.

Recommendations are given to present a unified user interface. Until Ver.1.00E, explanations were required when recommendations were not followed, but from ver.2.00EU recommendations mean just that: "recommendations".


Standardised

Indicates that in this case, the game was made based on past games. Refer to these items if you experience problems.

Prohibited

This item cannot be used in an application (the reason is shown at the same time).

'Unmarked,' 'Required,' 'Recommended,' and 'Standardized' items are applicable to either the Sega Brand trademark (Sega Manufacturing or Sales trademark) or a third-party trademark.

For applications developed on versions of Dreamcast not commercially sold, such as test versions, refer to the test version development standards (keep them in view while developing).

The software creation standards for Dreamcast are simplified by the removal of some items from the Saturn Software Creation Standards (although they may seem larger due to the added Required/Recommended/etc. classifications).

Although there are few written standards, this is not intended to mean that one is free to do anything one might imagine.

We have designed these Software Creation Standards to provide as much flexibility as possible, so you are still free to create programs that are easy to use (user-friendly).
