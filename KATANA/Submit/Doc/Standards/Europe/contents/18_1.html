Dreamcast Software Development Standards/Ver. 2.00EU

18.1 Vibration Peripheral (Product name the same)

18.1.1 Vibration settings


 Required: Include an option to toggle the setting ON/OFF for operation of the vibration peripheral.

In the future, there is the possibility that vibration peripherals and controllers with similar functions will be sold, so this setting should be made selectable.

Required: Stop vibration with the vibration peripheral during pause or when transferring to the Main Unit Boot ROM.

In the rules up to now, whether or not to stop vibration of the vibration peripheral "during transfer to the Main Unit Boot ROM" was not specially regulated, but from this version, stopping the vibration on the application side is regulated.

Required: When saving to the memory card, stop vibration of Vibration Pack.

There is a very slight chance that vibration may cause a momentary power failure in the connector terminal, so stop vibration to avoid damaging the connection.

Recommended: When checking the usable controllers at the start of a game and a vibration peripheral is detected to be inserted, include a warning such as "The Vibration Peripheral is operable", or make it possible for the user to select if the vibration peripheral should be turned ON with a message such as "Make Vibration Peripheral operational? (YES/NO)".

18.2.1 Current Check of the Vibration Peripheral

Required: Check the current when using the vibration peripheral.

Refer to the program manual for information about checking the current.

If the current exceeds the regulated value, the vibration device connected to a low-numbered port should be enabled.

Recommended: When the current exceeds the regulated value, an ON/OFF setting for the peripheral should be included with the message "Which Vibration Peripheral should be made operational?"
