4. Soft Reset and Processing When the Disk Door is Opened

4.1 Soft Reset

The Dreamcast Main Unit does not include a hardware RESET button, so a player can reset only by a soft reset function or by turning the power off and on. The conditions of 4.1.1 should therefore be carefully observed.

4.1.1 Soft Reset Timing

Required: When the disk door is open, "Soft Reset" must be enabled except in cases of special processing such as executing save and load of the memory card.

The recommended item 17.3.1 deals with the special processes mentioned above.

4.1.2 Soft Reset Button Operation

Required: A soft reset must occur as soon as the START Button is pressed when the A + B + X +Y buttons are all held down together, except for racing controllers, gun controllers and the keyboard for which this operation is impossible or for controllers for which this operation is difficult.

Recommended: A soft reset must also occur if any other buttons are pressed while the A + B + X + Y buttons are all held down when the START button is pressed.

Required: For controllers such as racing controllers, gun controllers and the keyboard for which the soft reset operation of pressing the A + B + X + Y buttons and the START button simultaneously is difficult, make soft reset possible with the method outlined in 18. Development Standards for Peripherals and Software Keyboard or through another procedure displayed in a separate menu.

When soft reset is supported through a procedure in a separate menu, it doesn't matter if that menu cannot be displayed at all times due to the timing of the game, but the soft reset option should be clearly displayed on a suitable menu and made selectable for the user.

To make the Soft Reset option selectable to jump to from the title screen and Main Unit Boot ROM menu, refer to 4.1.4.

Support soft reset using one of the procedures, such as menu display, described above even for an application in which all the buttons are independent during the game (including the START button), and in which it is possible to simultaneously use buttons.

4.1.3 Soft Reset and Controllers (Control Ports)

Required: Soft Reset must be possible from all controllers (ports) in use.

As explained in section 2.1.1 Single User, Single Controller Applications, operation is disabled for controllers connected to ports that are not being used by the application, so as a matter of course, soft reset is also disabled for those controllers.

4.1.4 Soft Reset Execution

Refer to A or B below and jump to the appropriate section.

A: The disc contains a single application. Go to section (4.1.4.1).

B: The disc contains multiple applications on an omnibus system. Go to (4.1.4.2).

4.1.4.1 Simple Application Case

Required: Soft reset from the Title Loop must jump to the Main Unit Boot ROM Menu.

Required: Soft reset from the main game (after the application is started) must jump to the Title Screen.

The process for determining whether an application has started (= in a main game) is described in 6. Title Screen.

4.1.4.2 Omnibus System Case

Recommended: Soft reset from within each game should cause a jump to the Title Screen of the corresponding main game.

Recommended: Soft reset from the Title Loop of any title should cause a jump to the base Title Screen.

Recommended: Soft reset from the base Title Screen Title Loop should cause a jump to the Main Unit Boot ROM Menu screen.

If the preceding recommendations cannot be followed (e.g., if no base Title Screen exists), follow the soft reset requirement of 4.1.4.1.

For applications that require more than one disc, refer to the chapter 17. Changing Discs for Multi-Disc Applications.

4.2 Processing with the Disk Door Open

This section explains processing when the disk door is opened during the regular main game. In this version, 17. Changing Discs for Multi-Disc Applications can be applied when the user opens the disk door in order to switch discs.

4.2.1 Principles when the Disk Door is Opened

Required: The Main Unit Boot ROM menu (the multi-player screen in Saturn) appears when the Main Unit "OPEN button" is pressed and the disk door is opened, except for the exceptions shown in 4.2.2, 4.2.3, and 4.2.4.

Go to the Main Unit Boot ROM menu within 10 seconds of when the disk door is left open.

It is difficult to detect if the disk door is opened and closed immediately when the disc is not accessed. However, in this case as well, go to the Main Unit Boot ROM menu as quickly as possible (detection is gradual).

4.2.2 Exceptions when the Disk Door is Open

Required: For games with data that require the changing of disks, such as "Monster Farm", a manual user mode for opening and closing the disk door is accommodated, and the disk door can only be opened and closed from this mode. A warning is displayed requesting the user to change back to the original game disc when finished with this mode, and after the original game disc is inserted, "Disk Check" should be carried out.

This option is limited to the times when it is necessary to change disks for game elements.

Refer to section 17.4 Switching Discs During Game Play for multi-disc applications.

For reasons other than this (to insert another disc as BGM, etc.), opening the disk door is prohibited.

4.2.3 When the Disk Door is Opened during Saves

Required: When executing saves, do not go to the Main Unit Boot ROM main screen until that save is finished. (This is to avoid losing the file being saved). Go to the main screen as soon as the save is finished.

4.2.4 When File Loading Fails from the Disc

Recommended: Do not go to the Main Unit Boot ROM main screen when displaying a message to the user that a file load error has occurred, as explained in section 16.2 When File Loading Fails from the Disc, even if the Main Unit disk door is opened.
