Dreamcast Software Development Standards/Ver. 2.00EU

1. Controller Button Functions

Required: Use the A button for "Accept" (perform operation) and the B button for "Cancel" (reject operation).

Allocation of "Accept" and "Cancel" to the respective A and B buttons is assumed, but that doesn't not mean the B Button must be removed from key configuration when the accept key is defined by pressing the button to allocate keys in key configuration.

Recommended: Use the A/B buttons first, and then the X/Y buttons.

Recommended: Set buttons to act when pressed instead of when released, except for special operations such as rapid fire in a shooting game.

1.1 Various Controller Exceptions

Recommended: Settings for the A and B buttons and 12.1 Pause Operation (START button) should conform to the arcade settings in a virtual arcade game if the original button settings still exist and a special controller is being used.

For standard controller operations, observe respective rules.

1.2 Cancel Operations

Recommended: An application that uses a lot of messages can have a Cancel button to speed up message processing.

Recommended: An application that uses voice can have a Voice Cancel button.

Recommended: An application that has demo scenes other than in title loops (demos that use movies or polygons) can use a Demo Cancel button. (Except when demos are required to advance the game.)

Standardised: (e.g., in "The Great Sakura War") The A button causes rapid message processing, the B button cancels voice and rapid message sending, and the START button cancels movies.

1.3 Analogue Direction Keys, Digital Direction Buttons

Recommended: Even games that mainly use the analogue keys should allow digital selections, such as of mode, by the digital direction buttons.
