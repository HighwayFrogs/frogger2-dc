18.2 Software Keyboard and Hardware Keyboard (Product name "Dreamcast Keyboard")

Note: Attach the hardware keyboard as the auxiliary device of the standard controller.

In this version, the hardware keyboard is not a type of controller; like the Saturn keyboard, the standard controller A button is not assigned to "Z" in specs.

Operation by hardware keyboard is limited to key input. All other operations must be done by a controller connected to a separate port.

Because the hardware keyboard does not have any expansion sockets, a memory card cannot be inserted into it.

Required: When the hardware keyboard is supported in an application, the software keyboard (or character input screen) should be displayed as an option and the user should be able to input by the software keyboard when the hardware keyboard is not connected.

A software keyboard is enabled when a hardware keyboard is supported by the application.

Required: During hardware keyboard input, allow the user to distinguish which input mode is currently being used.

When displaying the software keyboard screen and input palette screen, the user should know which input is currently possible.

18.2.2 Software and Hardware Keyboard Operation


 Recommended: Base software keyboard layout on the layout of "Dreamcast Passport 2.0", which is this company's standard application in the Main Unit.

Required: Use the special keys on the hardware keyboard, S1 and S2, as the left and right triggers, respectively.

Required: Follow the key input rules below for inputting the following items using operation of the software and hardware keyboards.

It is not necessary to allocate all the functions defined below.

- B: Hardware Keyboard Operation

Unifying key names

One key on the screen has one assigned name

example: "Enter" position key
- "Enter" key (name as printed)

One key on the screen is assigned in a combined upper and lower field (two names)

example: "Hiragana" position key
- "hiragana" key: indicates hiragana when pressed by itself
- "katakana" key: indicates katakana when pressed together with the SHIFT key

One key on the screen is assigned in a combined upper, lower, left and right field (2-4 names)

example: "3" position key
- "3" key: indicates 3 when pressed by itself during alphanumeric input
- "#" key: indicates # when pressed together with the SHIFT key during alphanumeric input
- "a (full)" key: indicates hiragana a when pressed by itself during kana input
- "a (half)" key: indicates hiragana a when pressed together with the SHIFT key during kana input

One key on the previous screen is assigned with one name

example: "hiragana" position key
- "alphabet" key: indicates letter input when pressed together with the ALT key.
